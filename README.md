# ربات بازنشر و ترجمه‌ی کانال — RadioBulletin

پست‌های کانال انگلیسی را کامل برمی‌دارد (با همه‌ی عکس‌ها/ویدیوها و آلبوم‌ها)، با هوش مصنوعی به فارسیِ روان و عامه‌پسند ترجمه می‌کند، آی‌دی انگلیسی را حذف و امضای کانال فارسی را ته پست اضافه می‌کند، در صورت تمایل روی عکس/ویدیو واترمارک می‌زند، و در کانال فارسی منتشر می‌کند — همه روی **GitHub Actions** و مستقل از اینترنت خودت.

---

## ⚠️ سه نکته‌ی مهم

۱) **حتماً اکانت تلگرام دومِ مجزا** برای ربات بگذار (نه اکانت اصلیت)؛ لاگین مکرر از سرور گیت‌هاب ممکن است اکانت را محدود کند. این اکانت دوم فقط باید عضو کانال انگلیسی باشد و در کانال فارسی حق ارسال داشته باشد.

۲) برای زمان‌بندیِ مطمئن از **cron-job.org** استفاده می‌کنیم (بخش پایین)، چون زمان‌بندی داخلی گیت‌هاب اغلب با تأخیر اجرا می‌شود.

۳) مدل پیش‌فرض `openai/gpt-4.1` است که بین مدل‌های گیت‌هاب بهترین کیفیت فارسی را دارد. سقف رایگانش محدود است؛ اگر پر شد، ربات خودکار به مترجم گوگل سوییچ می‌کند تا پستی از دست نرود.

---

## مرحله ۱ — ریپازیتوری
یک ریپوی **Private** بساز و این فایل‌ها را داخلش بگذار.

## مرحله ۲ — API_ID و API_HASH
با همان اکانت دوم وارد https://my.telegram.org شو ← API development tools ← اپ بساز ← `api_id` و `api_hash` را بردار.

## مرحله ۳ — ساخت رشته‌ی سشن
روی کامپیوتر خودت یک بار:
```bash
pip install telethon
python login.py
```
اطلاعات اکانت دوم را وارد کن؛ رشته‌ی طولانی خروجی را کپی کن (مثل رمز است، منتشرش نکن).

## مرحله ۴ — Secrets و Variables
ریپو → **Settings → Secrets and variables → Actions**

**Secrets:**
| نام | مقدار |
|---|---|
| `API_ID` | api_id |
| `API_HASH` | api_hash |
| `TG_SESSION` | خروجی login.py |
| `GH_MODELS_TOKEN` | اختیاری؛ Fine-grained PAT با دسترسی `models: read` |

**Variables:**
| نام | مقدار نمونه |
|---|---|
| `SOURCE_CHANNEL` | `@english_channel` یا `-100xxxx` |
| `TARGET_CHANNEL` | `@RadioBulletin` |
| `SOURCE_USERNAME` | یوزرنیم انگلیسی بدون @ |
| `WATERMARK` | `off` (برای روشن‌کردن: `on`) |

## مرحله ۵ — روشن کردن و تست
تب **Actions** → workflow «mirror» را یک‌بار دستی Run کن. اجرای اول فقط نقطه‌ی شروع را ثبت می‌کند (آرشیو قدیمی را نمی‌فرستد) و از پست بعدی آینه‌سازی می‌کند.

---

## ⏱️ زمان‌بندی با cron-job.org (روش اصلی)

اول یک توکن بساز: GitHub → Settings → Developer settings → **Fine-grained personal access token** با دسترسی **Actions: Read and write** روی همین ریپو. (یا توکن کلاسیک با اسکوپ `repo` و `workflow`.)

در https://cron-job.org یک Cronjob جدید بساز:
- **URL:**
  `https://api.github.com/repos/<USERNAME>/<REPO>/actions/workflows/mirror.yml/dispatches`
- **Method:** `POST`
- **Headers:**
  - `Accept: application/vnd.github+json`
  - `Authorization: Bearer <YOUR_TOKEN>`
  - `X-GitHub-Api-Version: 2022-11-28`
- **Request body:** `{"ref":"main"}`  (اگر شاخه‌ی اصلی‌ات `master` است همان را بگذار)
- **Schedule:** هر ۱۰ یا ۳۰ دقیقه

اگر درست باشد، گیت‌هاب پاسخ **204 No Content** می‌دهد و در تب Actions اجرای جدید می‌بینی. (یک `schedule` نیم‌ساعته هم به‌عنوان پشتیبان داخل workflow هست؛ اجراهای هم‌زمان به‌خاطر قفل concurrency و ذخیره‌ی state، نه پست تکراری می‌سازند نه گم‌شده.)

## 🖼️ واترمارک روی عکس و ویدیو

۱) لوگوی خودت را با نام `logo.png` داخل پوشه‌ی `watermark/` بگذار (PNG شفاف بهترین است؛ بهتر است متن «رادیو بولتن» را همان داخل تصویر طراحی کنی تا فارسی بی‌نقص رندر شود).
۲) متغیر `WATERMARK` را روی `on` بگذار.
۳) عکس‌ها با Pillow و ویدیوها با ffmpeg واترمارک می‌شوند.

تنظیمات (در فایل `mirror.yml`): `WM_POSITION` (گوشه)، `WM_SCALE` (اندازه نسبت به عرض، پیش‌فرض ۰٫۱۸)، `WM_OPACITY` (شفافیت ۰ تا ۱).

> توجه: واترمارک ویدیو یعنی ری‌انکد شدن ویدیو که کندتر و سنگین‌تر است؛ برای کانال خبری با ویدیوهای زیاد، ممکن است اجراها طولانی شوند.

## امضای ته پست
ته هر پست (حتی پست‌هایی که فقط عکس/ویدیو هستند و متن ندارند) این امضا اضافه می‌شود:
`@RadioBulletin | رادیو بولتن`
و اگر یوزرنیم کانال انگلیسی داخل متن باشد، با `@RadioBulletin` جایگزین می‌شود.

## تضمین عدم گم/تکرار شدن
شناسه‌ی آخرین پست در `state/last_id.txt` ذخیره و بعد از هر ارسال موفق در ریپو commit می‌شود؛ پس اگر خطایی رخ دهد، اجرای بعدی از همان‌جا ادامه می‌دهد.

## اجرای دائمی روی سرور (جایگزین سریع‌تر)
همین `bot.py` را می‌توان با کران لینوکسی هر ۲ دقیقه روی یک سرور کوچک اجرا کرد؛ چون آی‌پی ثابت است ریسک محدودیت اکانت بسیار کمتر است:
```bash
*/2 * * * * cd /path/to/project && /usr/bin/python3 bot.py >> bot.log 2>&1
```

## امنیت
ریپو را Private نگه دار؛ `*.session` و `.env` را commit نکن (در `.gitignore` هست)؛ توکن‌ها فقط در Secrets.
